Why increased focus on tools?

  • OMERO.server API offers a lot of functionality that is not yet used by clients
  • New clients can add many new features for users without needing any changes to OMERO.server
  • More flexible release process for decoupled clients...

OMERO.server release process

  • Releases take OME team a whole day
  • OMERO upgrade requires server downtime
  • Large application with many parts in a single release:
    • Clients tightly coupled to OMERO.server
    • e.g. Bug-fixes or new features in OmeroPy or OmeroWeb require a full OMERO release

Advantages of decoupled clients
For Developers:

  • Prototypes can be developed and installed without breaking anything
  • More flexible release cycles
  • Release takes OME team ~1 hour

Advantages of decoupled clients
For Users:

  • Simpler for Users to upgrade: no server downtime
  • Only install the apps you want
  • Can install new Apps with an older OMERO.server
  • Installed Apps integrate well with existing clients, e.g:
    • Web: Centre panel
    • Web: Open with...

Disadvantages of decoupled clients

  • Discoverability: Adding feature pages to website
  • More steps to install "everything"
  • Devs: More repos and testing infrastructure to maintain

Testing of decoupled clients

  • Use omero-test-infra
  • Docker-composed OMERO server for testing
  • Testing scripts run by Travis CI on every GitHub PR
  • For example: OMERO.figure .travis.yml
